# 변경 이력

ClipCatcher의 사용자에게 영향을 주는 변경 사항을 버전별로 기록합니다.

## [2.0.7] - 2026-07-13

### 수정

- YouTube 다운로드가 MP4 컨테이너 안에 `AV1 + Opus`로 저장되어 Final Cut Pro에서 열리지 않을 수 있던 문제를 수정했습니다.
- 선택한 화질 범위 안에서 Final Cut Pro 호환성이 높은 `H.264(avc1) + AAC(m4a)` 스트림을 우선 사용합니다.
- YouTube 호스트 판별을 정확한 도메인 기준으로 강화하고, 치지직 포맷 선택에는 영향을 주지 않도록 분리했습니다.

### 검증

- 360p, 720p, 1080p YouTube 포맷 선택에서 `H.264 + AAC` 조합을 확인했습니다.
- 실제 1080p 5초 구간을 다운로드해 `H.264 High`, `yuv420p`, `AAC-LC`, MP4 출력을 확인했습니다.
- 치지직 포맷 비회귀 테스트와 패키지 스모크 테스트를 통과했습니다.
- Windows GitHub Actions가 단위 테스트와 소스 스모크 테스트를 통과한 경우에만 패키지를 빌드하도록 강화했습니다.

## [2.0.6] - 2026-07-11

### 추가

- 다운로드 센터 상단에 현재 설정된 `저장 폴더 열기` 버튼을 추가했습니다.

### 개선

- 다운로드 항목 너비가 560px보다 좁아지면 썸네일을 숨기고 작업 버튼을 축소해 핵심 정보와 동작이 계속 보이도록 개선했습니다.
- 긴 영상 제목이 고정 폭에 막히지 않고 카드 너비에 맞춰 줄바꿈되도록 개선했습니다.
- 완료 항목의 `파일 열기` 버튼은 좁은 화면에서 `열기`로 표시해 작은 창에서도 접근할 수 있습니다.

### 수정

- 작은 창에서 완료된 다운로드의 파일 열기 동작이 화면 밖으로 잘리던 문제를 수정했습니다.
- macOS 앱 정보에 배포 버전이 `0.0.0`으로 표시되던 패키징 메타데이터를 `2.0.6`으로 수정했습니다.
- macOS 빌드 스크립트가 임시 서명 검증, 패키지 스모크 테스트, 배포 ZIP 생성을 한 번에 수행하도록 수정했습니다.

### 검증

- Python 문법 검사와 소스 스모크 테스트를 통과했습니다.
- 420px 및 720px 너비에서 다운로드 카드의 반응형 표시를 회귀 테스트했습니다.
- 태그 배포 시 Windows 패키지 스모크 테스트를 GitHub Actions에서 수행합니다.

이 파일은 `2.0.6`부터 관리합니다. 이전 릴리즈 기록은 [GitHub Releases](https://github.com/ThankyouJerry/ClipCatcher/releases)에서 확인할 수 있습니다.
